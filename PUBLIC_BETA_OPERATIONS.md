# Public Beta Operations

This file explains the calm release rule for the current bounded public beta surface.

## The Loop

1. identify one bounded issue
2. fix the smallest real surface
3. rerun the nearby proofs
4. update the public docs if the outward story changed
5. release only after the bounded truth still holds

## Why This Matters

Queenshift should become more trustworthy by tightening the bounded lane, not by widening claims every time a proof turns green.

## Current Boundary

1. public beta visibility is real
2. the broad general-use answer is still `NO`
3. release work should preserve that distinction
