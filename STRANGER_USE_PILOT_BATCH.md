# Stranger-Use Pilot Batch

This file records the bounded stranger-style pilot reading that informed the public release surface.

## What It Showed

1. the project needs a very explicit first path
2. provider setup is still the first likely live blocker
3. guided demo and disposable demo are the safest opening lanes
4. source-checkout and stranger-first surfaces should not be blurred together

## Why It Is Public

This is not here as a marketing artifact.

It is here so the public repo shows that the install and support story was shaped by bounded usability evidence rather than guesswork.
