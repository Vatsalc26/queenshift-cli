# Queenshift Public Docs

This folder holds the main supporting docs for the public Queenshift repo.

Current public-doc entry points:

1. `../README.md` for the product overview
2. `../QUICKSTART.md` for the recommended first run
3. `../AUTHORS.md`, `../CITATION.cff`, `../.zenodo.json`, `../CHANGELOG.md`, `../CONTRIBUTING.md`, `../CODE_OF_CONDUCT.md`, `../SECURITY.md`, and `../LICENSE` for the release and governance surface
4. `../SUPPORTED_INSTALL_SURFACES.md`, `../CONTRIBUTOR_SOURCE_CHECKOUT.md`, and `install.md` for install status and source-checkout boundaries
5. `providers.md` and `../PROVIDER_SETUP_GUIDE.md` for provider setup and fail-closed rules
6. `task-families.md`, `../TASK_CORPUS.md`, `../LANGUAGE_PACKS.md`, and `../LANGUAGE_RELIABILITY_MATRIX.md` for current task families and language coverage
7. `evidence.md`, `../GENERAL_USE_READINESS_DECISION.md`, and `../GENERAL_USE_RELEASE_CANDIDATE_GATE.md` for the current bounded evidence and broader-readiness reading
8. `../OWNER_OVERSIGHT_GUIDE.md`, `../SUPPORT_ISSUE_INTAKE.md`, and `../BOUNDED_1_0_SUPPORT_RUNBOOK.md` for owner/support surfaces
9. `../VERIFICATION_CATALOG.md` for the current proof loops and verification reading
10. `../.github/ISSUE_TEMPLATE/` plus `../.github/pull_request_template.md` for public bug, task-family, and PR intake

These docs stay stranger-first, contributor-friendly, and bounded around the current public release surface.
