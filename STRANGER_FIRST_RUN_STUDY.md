# Stranger First-Run Study

This file records the main first-run friction themes that shaped the current public docs.

## Main Lessons

1. audience split has to stay explicit
2. provider setup is the first likely confusing blocker
3. the calmest first task should stay tiny and explicit
4. demo and replay surfaces help more than broad marketing language

## Why It Matters Publicly

This repo should look careful because it learned from first-run friction instead of pretending that setup is already effortless for everyone.

## Current Boundary

1. the source-checkout route is real
2. the calmer normal-user install story is still incomplete
3. the project should stay honest about that
