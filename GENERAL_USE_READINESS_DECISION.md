# General-Use Readiness Decision

## Current Decision

`NO`

Queenshift is public, serious, and useful in bounded lanes.

It is not yet honest to call it a broad general-use coding CLI.

## Why The Answer Is Still No

1. benchmark completeness is still limited
2. install breadth is still limited
3. larger-repo breadth is still limited
4. some capability surfaces remain intentionally asymmetric

## What Can Be Claimed Honestly

1. bounded coding CLI
2. governance-first architecture
3. artifact-backed review, replay, and incident surfaces
4. strong small-scope story
5. explicit separation between shipped `swarmengine` and experimental `queenbee`

## What Should Not Be Claimed Yet

1. broad general-use autonomous coding
2. open-ended repo-wide refactors
3. broad packaged cross-platform parity
4. default-engine switch to QueenBee
5. benchmark-victory language

## Why This File Exists

This file keeps the public story disciplined.

The project should look serious because it is honest, not because it inflates the current state.
